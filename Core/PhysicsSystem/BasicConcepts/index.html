力与运动

Forces

Movement

Explicit Euler's Method

因为游戏中的 $\bigtriangleup t$ 远大于现实世界中的普朗克时间，所以可能会导致能量变多。

Implicit Euler's Method

会导致能量变少（可以接受）。

解隐式欧拉法需要用追赶法解矩阵。

Semi-implicit Euler's Method

需要假设力的大小与位置无关，再用未来的速度去计算未来的位移。

碰撞检测

Broad Phase

初筛，将所有物体都用 AABB 包围盒包裹，判断这些包围盒是否相交。

BVH Tree

更新成本低，适合动态场景。

Sort and Sweep

分别按照 X，Y 和 Z 三个方向排序所有包围盒的边界，根据边界的交叉粗略判断包围盒是否相交，适合静态场景。

Separating Axis Theorem

如果两凸包不相交，则一定能用任一凸包的一条边将两个凸包分离至两侧。

3D 情况下就要用面去分离。

碰撞解决

Applying Penalty Force

直接用力将两个物体弹开。

Solving Constraints

在约束条件限制下用小步迭代法找冲量解。

之所以是冲量而不是力，主要是考虑到冲量可以直接转换成速度去计算。

为了防止死循环，还会设置迭代精度以及迭代步长等超参数。

效率，准确性与确定性

Sleeping

将场景中的对象分成好几个 Island，如果一些 Island 一段时间内没有变化，则让它们 Sleeping。

Continuous Collision Detection

当一个快速运动的对象碰撞一个比较薄的物体时容易发生错误而直接穿过去（前后两次计算都没有发现碰撞）。

可以先预估一个安全距离，在这个距离外时正常计算，在这个距离内则需要降低迭代步长，提高计算精度。

Deterministic Simulation

实现确定性模拟，就意味着如果两台电脑的输入相同，那么它们显示的内容也会相同，这也是联机游戏的基础。
